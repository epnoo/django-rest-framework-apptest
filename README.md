# 🎬 Django REST Framework - Movie API

A RESTful API built using Django REST Framework (DRF) to manage Movies, Directors, and Genres data. This project features full CRUD operations as well as custom endpoints for importing and exporting data in Excel (`.xlsx`) format.

## ✨ Key Features
* **CRUD Operations:** Fully manage `Movie`, `Director`, and `Genre` data through the built-in DRF interface.
* **Database Integrity Protection:** Custom error handling when deleting master data (Directors/Genres) that still have relations with Movie data. It returns a clean `400 Bad Request` response instead of a server crash (Safe implementation of `DO_NOTHING`).
* **Export to Excel:** Download the entire list of movies along with their relational data into an `.xlsx` format.
* **Import from Excel:** Add dozens of movie records at once via Excel file upload. It includes a custom HTML interface (`test_upload.html`) to simulate real-world frontend integration.

## 🛠️ Technologies Used
* **Backend:** Python 3, Django, Django REST Framework
* **Database:** PostgreSQL (Tested using DBeaver)
* **Dependency Manager:** Poetry
* **Additional Libraries:** `openpyxl` (for Excel file processing)

## 🚀 Installation Guide (Development Setup)

Follow the steps below to run the project locally on a new machine.

### 1. Clone the Repository
```bash
git clone [https://github.com/epnoo/django-rest-framework-apptest.git](https://github.com/epnoo/django-rest-framework-apptest.git)
cd django-rest-framework-apptest
```

### 2. Install Dependencies (using Poetry)
This project uses [Poetry](https://python-poetry.org/) to ensure consistent dependency versions. Run this command to install all required libraries:
```bash
poetry install
```

### 3. Database Configuration
1. Open PostgreSQL / DBeaver.
2. Create a new empty database (ensure the name matches the configuration in `settings.py`).
3. Make sure the *username* and *password* in `settings.py` match your local PostgreSQL credentials.

### 4. Database Migration
Apply the table structures to the newly created database:
```bash
poetry run python manage.py migrate
```

### 5. Run the Server
```bash
poetry run python manage.py runserver
```
The API is now accessible at: `http://127.0.0.1:8000/api/`

## 🧪 Testing & Demo Scenario

To fully test the functionalities, you can follow this workflow:

1. **Input Master Data (Manual via DRF UI)**
   * Open your browser and navigate to `http://127.0.0.1:8000/api/directors/` and `http://127.0.0.1:8000/api/genres/`.
   * Add at least 3 Directors and 3 Genres. Note the IDs generated by the database.
2. **Test Excel Import Feature**
   * Open *File Explorer* and double-click the `test_upload.html` file located in the root directory.
   * On the web page, upload the `dummy_movies.xlsx` file provided in this repository.
   * Click **Upload & Import**. Hundreds of movie records will automatically be inserted into the database.
3. **Test Excel Export Feature**
   * Open your browser and access the endpoint: `http://127.0.0.1:8000/api/movies/export_excel/`
   * An `.xlsx` file containing all movie data will be downloaded automatically.
4. **Test Delete Validation (Integrity Error)**
   * Go to `http://127.0.0.1:8000/api/directors/1/` (Make sure Director ID 1 is already linked to a movie).
   * Click the **DELETE** button.
   * The system will reject the deletion and return a neatly formatted error message due to database relationship rules.
